# 数据字典与采集清洗规范

## 1. 数据字典
### 表结构定义
#### `themes` 表
- **描述**：存储主题信息。
- **字段**：
  - `id`（主键）：主题唯一标识。
  - `name`：主题名称。
  - `description`：主题描述。

#### `content` 表
- **描述**：存储具体内容（文字、图片等）。
- **字段**：
  - `id`（主键）：内容唯一标识。
  - `theme_id`（外键）：关联 `themes` 表。
  - `text`：文字内容。
  - `image_url`：图片链接。
  - `source`：数据来源。

## 2. 数据采集清洗规范
### 采集流程
1. **数据来源**：
   - API：如 TMDb、Unsplash、Google Books。
   - 爬取：遵守 robots.txt 和版权要求。
   - 手动导入：支持 CSV/JSON 格式。

2. **采集工具**：
   - 使用爬虫框架（如 Scrapy）。
   - 使用 API 客户端（如 Axios）。

### 清洗流程
1. **规范化**：
   - 字段标准化：主题、题材、语言、标签、来源 URL、版权状态。
   - 数据格式：统一编码（UTF-8）。

2. **清洗规则**：
   - 去重：移除重复内容。
   - 语言检测：确保内容语言符合主题要求。
   - 长度校验：文字内容不超过 60 字，图片大小不超过 2MB。
   - 敏感词过滤：移除不适宜内容。
   - 图片质量检测：确保分辨率和清晰度达标。

### 标注与审核
1. **标注**：
   - 添加主题标签、风格标签、关键词。
   - 标记来源和版权许可。

2. **审核**：
   - 人工抽检：随机抽取 10% 数据进行质量检查。
   - 评分机制：低分内容隔离，质控阈值≥0.8。

### 发布与更新
1. **发布**：
   - 生成只读数据库，分配版本号。
   - 记录变更日志。

2. **更新**：
   - 周期性更新（每周一次）。
   - 紧急修复即时发布。
