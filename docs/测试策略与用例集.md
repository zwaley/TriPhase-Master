# 测试策略与用例集

## 1. 测试策略
### 测试范围
- **单元测试**：
  - 自动生成内容的算法。
  - 日期计算逻辑。
  - 前端组件的渲染和交互。

- **集成测试**：
  - 前端与后端 API 的交互。
  - 数据库的读写操作。

- **E2E 测试**：
  - 用户从输入到导出的完整流程。
  - 离线模式的功能验证。

### 测试目标
- 确保核心功能的正确性和稳定性。
- 验证系统在不同场景下的兼容性和性能。
- 提高代码覆盖率，目标：语句 80%、函数 70%、分支 60%。

## 2. 测试用例集
### 用例 1：自动生成内容
- **前置条件**：
  - 数据库中有可用的内容。
  - 用户选择了主题和日期。
- **测试步骤**：
  1. 打开生成器页面。
  2. 选择主题（如“电影”）。
  3. 点击“自动生成”按钮。
- **预期结果**：
  - 显示生成的日签内容。
  - 内容包括文字和图片，符合主题。

### 用例 2：日签导出
- **前置条件**：
  - 已生成日签内容。
- **测试步骤**：
  1. 点击“导出”按钮。
  2. 选择保存路径。
- **预期结果**：
  - 成功下载 PNG 格式的图片。
  - 图片清晰，内容完整。

### 用例 3：离线模式
- **前置条件**：
  - 浏览器已缓存离线内容。
  - 网络断开。
- **测试步骤**：
  1. 打开生成器页面。
  2. 点击“自动生成”按钮。
- **预期结果**：
  - 显示离线生成的日签内容。
  - 不出现网络错误提示。
